# 🏗️ Data Warehouse & Analytics Engineering — Practice Repository

> *This is just the beginning. Every week, a new warehouse. Every month, new tools. Watch this space.*

## 📌 What is this repo?

This is a living, evolving portfolio of **data warehouse builds** — one per week, increasing in complexity — documenting my hands-on journey from ESG data professional to **Analytics Engineer**.

Each warehouse is built from scratch using real-world inspired datasets and scenarios. You’ll find SQL scripts, Jupyter notebooks, data models, transformation logic, and documentation — all version-controlled and progressively more sophisticated as I layer in new tools and techniques.

## 👤 A bit of context

I’m a Senior Associate in **ESG Reporting & Assurance** field, with a background spanning sustainability analytics, Scope 3 emissions, and supply chain data. Over the past few years I’ve self-developed skills in **Python, SQL, Power BI, and Power Automate**, applying them to automate and improve ESG data pipelines — work that’s earned recognition at the CEO level at GSK.

Now I’m taking that foundation and building something bigger: a deliberate, structured transition into **Analytics Engineering**, combining deep ESG domain expertise with genuine data engineering capability.

This repo is the proof of work.

## 🗂️ What you’ll find here

Each folder represents a standalone warehouse project, tagged by week and skill level. Expect to see:

- **Dimensional modelling** — star schemas, snowflake schemas, slowly changing dimensions
- **SQL transformations** — from raw to clean to analytics-ready
- **Snowflake** — warehouses, stages, Snowsight dashboards
- **dbt** — models, tests, documentation, lineage graphs
- **Python / Jupyter** — EDA, ingestion scripts, automation
- **ESG-flavoured datasets** — emissions, energy, water, supply chain, assurance
